# 🤝 Contributing to AttenMo

Thank you for your interest in AttenMo! This document explains how you can contribute to the project.

> **⚠️ Important:** AttenMo is a **proprietary project**. Direct code contributions via pull requests are **not accepted** at this time. However, there are several valuable ways you can help improve the platform.

---

## 🛠️ How You Can Contribute

### 🐛 Bug Reports
Found a bug? Please report it so we can fix it.

**Before reporting:**
- Check [existing issues](https://github.com/shivam238/attendance-system/issues) to avoid duplicates.
- Confirm the bug exists on the **latest version** of the web app or APK.

**When reporting, include:**
- Clear title and description of the issue
- Steps to reproduce (numbered, step-by-step)
- Expected vs. actual behavior
- Screenshots or screen recordings if applicable
- Device/browser/OS info (e.g., Chrome 125 / Android 14 / Windows 11)

> Open a [new issue](https://github.com/shivam238/attendance-system/issues/new) with the `bug` label.

---

### 💡 Feature Requests & Suggestions
Have an idea that would make AttenMo better?

- Open a [new issue](https://github.com/shivam238/attendance-system/issues/new) with the `enhancement` label.
- Describe the use case clearly — *why* is this feature needed?
- If possible, suggest how it could work (UI mockup, flow description, etc.).

---

### 🔐 Security Vulnerabilities
**Please do NOT open public issues for security vulnerabilities.**

Refer to our [Security Policy](./SECURITY.md) for responsible disclosure instructions.

---

### 📝 Documentation Improvements
Spotted a typo, unclear wording, or outdated info in the docs?

- Open an issue with the `documentation` label describing the problem.
- You may also reach out via [WhatsApp Community](https://chat.whatsapp.com/GDOjvKK7nxGFvQ7NSrQNiG).

---

### ⭐ Star & Share
The simplest contribution — **star the repo** and share AttenMo with classmates, educators, and institutions who could benefit from it!

---

## 📋 Issue Guidelines

| Do ✅ | Don't ❌ |
|---|---|
| Use a clear, descriptive title | Submit vague reports like "it doesn't work" |
| Provide reproduction steps | Skip steps or assume the developer knows your setup |
| One issue per report | Bundle multiple unrelated bugs in one issue |
| Be respectful and constructive | Use rude or demanding language |
